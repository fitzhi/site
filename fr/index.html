Bienvenue sur le site de Fitzhì !
la synthèse technique...

Fitzhì est une startup Bordelaise, à l'origine d'une solution open-source éponyme. Fitzhì établit une synthèse technique globale d'un système d'information, d'un portefeuille d'applications ou plus simplement d'un progiciel. « Où en sommes nous ? » A cette question simple, Fitzhì veut répondre, simplement, avec un minimum d'efforts. Cette solution adresse deux dimensions des DSI : les applications et les équipes autour d'elles.
    - Une écurie de formule 1, c'est une voiture, et c'est un paddock.

Fitzhì se propose de répondre à ces questions :
  • Notre SI s'est-il techniquement amélioré depuis l'année dernière ?
  • Le turnover de la société dégrade-t'il notre maitrise des applications ?
  • Quelles sont les personnes clés de ma DSI, visibles ou cachées, de par leur connaissance des applications ?
  • De quelles compétences risque-t'on de manquer pour couvrir l'écosystème de nos applications ?
  • Peut-on évaluer techniquement la DSI de l'entreprise dont nous venons de prendre le contrôle ?
  • Quel niveau de couverture des écosystèmes de notre portfolio est opéré par nos équipes IT ?
Plusieurs exemples de synthèses vous sont présentés ci-dessous :

LE PATCHWORK DU PORTFOLIO

Ce patchwork représente toutes les applications d'une DSI.

Dans cette exemple, nous avons créé un patrimoine factice à partir de tous les projets open-source dont dépend Fitzhì.

Chaque tuile représente une application. Sa dimension est proportionnelle au nombre de lignes. Sa couleur figure le niveau de risque mesuré par Fitzhì sur le plan Staffing, Qualité du code et Conception.

la ligne d'horizon du portfolio

Cette ligne d'horizon qui s'élève représente l'historique d'un portefeuille depuis son premier commit jusque la date du jour.
C'est l'histoire de vos projets, du point de vue du code source, qui s'élève ici.

Chaque « bâtiment » est un projet qui grimpe à la vitesse du nombre de lignes de code.
La couleur varie dans le temps suivant le risque Staffing pour le projet.

la couverture des compétences

Cette carte proportionnelle represente la couverture des compétences sur l'ensemble des projets enregistré dans Fitzhì.

Il évalue le volume des écosystèmes présents dans votre DSI et les équipes actives qui sont prêtes à les maintenir.

  • Chaque tuile represente un ecosystème (avec un maximum de 10 tuiles dans cette exemple), en proportion de son importance dans le portfolio.
  • Chaque tuile a une couleur qui figure the niveau de risque evaluaé pour cet écosystème.

    « Vert » est sous control.

    « Rouge » est à risque.

    Dans cet exemple, Javascript est mieux couvert que Java. Typescript quant à lui, est à risque modéré.
  • Il est possible d'inclure, ou pas, dans l'analyse les developpeurs externes.
  • Il est possible aussi de filtrer les developpeurs par seniorité (du 1 à 5).

le diagramme sectoriel composite

Le diagramme sectoriel résume sur un seul graphique, la qualité du portefeuille sur les 3 dimensions qui sont actuellement étudiées par Fitzhì :

  • la couverture des compétences vis à vis des projets
  • la qualité du code
  • la conception des applications

Par conséquent, le diagramme est divisée en 3 parties, et chaque partie a 4 secteurs possibles de taille différente.

les tableaux de bord « Project »

Chaque projet déclaré dans Fitzhì, se voit attribuer une une médaille, qui regroupe sa synthèse technique.

Cette médaille regroupe 3 indicateurs :

  • Le niveau de couverture du code justifiée par l'actuelle équipe de développement
  • L'évaluation technique de l'application, selon la perspective Sonar
  • L'évaluation technique de l'application à dire d'experts

le tableau de bord Staffing

Les sections vertes sont des packages avec suffisamment de développeurs actifs.

Les sections rouges sont les déserts. Il n'y a plus personne pour maintenir le code.

Notre niveau de maîtrise avec nos effectifs est-il suffisant pour sécurisuer le suivi de cette application? L'efficacité opérationnelle est souvent tributaire du turnover.

Disposez-vous d'une équipe crédible, avec les compétences, et les connaissances requises, qui puisse assurer le maintien en condition opérationnelle de cette application ?

Vous devez être en mesure de répondre à des questions telles que

  • Qui a travaillé sur ce projet ? Sont-ils encore là ?
  • Avons-nous quelqu'un de disponible pour fixer ce problème dans cette application archaïque ?
  • La maîtrise technique de l'équipe actuelle couvre-t-elle toutes les parties du projet ? Ou prenons-nous des risques quelque part ?

Vous pouvez naviguer dans l'arborescence du project en clickant sur les sections du diagramme.

le tableau de bord Sonar

Fitzhì connecte chaque application avec sa perspective Sonar™. Il en tire une note unique, basée sur le produit vectoriel des métriques Sonar pondérées par leurs poids relatifs.

Notre application Fitzhì avait une note de 52, il y a bien longtemps dans le passé....

le tableau de bord Audit

Aucun outil automatisé ne peut pas évaluer intégralement une application.

Plus ou moins régulièrement, des experts sont mandatés pour les auditer. Ils les évaluent sur des thèmes qui vous apparaissent comme clés, et significatifs pour celles-ci.

Dans l'exemple ci-dessous, l'organisation générale, le processus de construction et la documentation ont été considérés comme importants pour couvrir l'évaluation.
