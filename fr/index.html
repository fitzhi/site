Bienvenue sur Fitzhì !
Supervision technique de la DSI.

Fitzhì est une société Bordelaise, éditeur de la solution open-source éponyme : Fitzhì©.
Notre solution se propose de construire la synthèse technique de votre système d'information, d'un portefeuille d'applications ou plus modestement d'un progiciel. Notre crédo est d'être synthétique, et dynamique. Les analyses de Fitzhì adressent aussi bien la qualité des développements que le niveau technique des équipes.
Fitzhì est une solution destinée aux DSI, CTO, responsables de centres de services ou responsables de départements.

Fitzhì veut répondre à des questions comme :
  • Notre SI s'est-il techniquement amélioré depuis l'année dernière ?
  • Le turnover de la société dégrade-t'il notre maitrise des applications ?
  • Quelles sont les personnes clés de ma DSI, visibles ou cachées ?
  • De quelles compétences risque-t'on de manquer pour couvrir l'écosystème de nos applications ?
  • Peut-on évaluer techniquement la DSI de l'entreprise dont nous venons de prendre le contrôle ?
  • Quel niveau de couverture des écosystèmes de notre portfolio est-il sécurisé par nos équipes ?
  • S'il fallait nous donner une note générale ? Une moyenne générale ? Quelle serait-elle ?
Plusieurs exemples de synthèses vous sont présentés ci-dessous :

LE PATCHWORK DU PORTFOLIO

Ce patchwork représente toutes les applications d'une DSI.

Dans cet exemple, nous avons créé un patrimoine fictif, à partir de tous les projets open-source utilisés par Fitzhì.

Chaque tuile représente une application. Sa dimension est proportionnelle au nombre de lignes de code du projet. La couleur illustre l'évaluation composite telle que mesurée par Fitzhì. Cette évaluation combine les dimensions Équipe, qualité du code et conception.

la ligne d'horizon du portfolio

Cette ligne d'horizon qui s'élève représente l'historique d'un portefeuille de projets depuis son premier commit jusque la date du jour.
C'est l'histoire des projets, du point de vue du code source, qui s'élève ici.

Chaque « bâtiment » est un projet qui grimpe à la vitesse du nombre de lignes de code.
La couleur qui évalue dans le temps représente le risque Staffing du projet.

la couverture des compétences

Cette carte proportionnelle (en anglais treemap) représente la couverture des compétences sur l'ensemble des projets enregistrés dans Fitzhì.

Ce diagramme présente le volume des écosystèmes présents dans votre DSI, et les équipes actives qui sont disponibles pour les maintenir. Ces développeurs ne connaissent pas nécessairement ces applications, mais ils sont compétents dans l'écosystème.

  • Chaque tuile représente un ecosystème (avec un maximum de 10 tuiles dans cet exemple). Sa taille est proportionnelle à son importance dans le portfolio. Dans cet exemple, Java et JavaScript sont très majoritaires.
  • Chaque tuile porte aussi une couleur qui illustre le niveau de risque evalué pour cet écosystème.

    « Vert » est sous control.

    « Rouge » est à risque.

    Dans cet exemple, Javascript est mieux loti que Java. On mesure un risque modéré sur Typescript.
  • Il est possible d'intégrer, ou pas, les développeurs externes dans l'évaluation globale.
  • Il est possible aussi de filtrer les developpeurs par seniorité (du 1 à 5).

le capital « compétences » de l'organisation

Ce graphe suit mensuellement le niveau global de compétences de votre organisation. Chaque développeur peut avoir 5 niveaux (★) de séniorité par compétence.

Le CEO de Fitzhì se résume dans le bandeau ci-dessous :

Il contribue donc par 24 ★, au capital « compétences » de l'organisation.

Cet indicateur combine le nombre de développeurs et leur niveau technique, comme la qualité de mouvement combine une vitesse et une masse.

le diagramme sectoriel composite

Le diagramme sectoriel (pie chart en anglais, camenbert en français) résume sur un seul graphique, la qualité du portefeuille sur les 3 dimensions qui sont actuellement étudiées par Fitzhì :

  • la couverture des compétences vis à vis des projets (Staff)
  • la qualité générale du code (Sonar)
  • le design général des applications (audit)

Le diagramme est donc divisé en 3 parties. Chaque partie a 4 secteurs possibles de taille différente qui classent les projets en 3 quartiers (niveaux) de risque. Le quartier gris quant à lui, est attribué aux projets sans évaluation.

les tableaux de bord « Projet »

Chaque projet déclaré dans Fitzhì, se voit attribuer une médaille, qui résume sa synthèse technique.

Cette médaille regroupe 3 indicateurs :

  • Le niveau de couverture du code justifié par l'équipe actuelle de développement
  • L'évaluation technique de l'application, selon la perspective Sonar
  • L'évaluation technique de l'application à dire d'experts

le tableau de bord Staffing

Les sections vertes sont des packages avec suffisamment de développeurs actifs.

Les sections rouges sont les déserts. Il n'y a plus personne pour maintenir le code.

Notre niveau de maîtrise avec nos effectifs est-il suffisant pour sécurisuer le suivi de cette application? L'efficacité opérationnelle est souvent tributaire du turnover.

Disposez-vous d'une équipe crédible, avec les compétences, et les connaissances requises, qui puisse assurer le maintien en condition opérationnelle de cette application ?

Vous devez être en mesure de répondre à des questions telles que

  • Qui a travaillé sur ce projet ? Sont-ils encore là ?
  • Avons-nous quelqu'un de disponible pour fixer ce problème dans cette application archaïque ?
  • La maîtrise technique de l'équipe actuelle couvre-t-elle toutes les parties du projet ? Ou prenons-nous des risques quelque part ?

Vous pouvez naviguer dans l'arborescence du project en clickant sur les sections du diagramme.

le tableau de bord Sonar

Fitzhì connecte chaque application avec sa perspective Sonar™. Il en tire une note unique, basée sur le produit vectoriel des métriques Sonar pondérées de leurs poids relatifs.

Notre application Fitzhì avait une note de 52, il y a bien longtemps dans le passé....

le tableau de bord Audit

Aucun outil automatisé ne peut évaluer intégralement, sous l'ensemble de ces aspects, une application. L'humain tient sa place. Plus ou moins régulièrement, des experts sont mandatés pour auditer. Ils choisissent des thèmes qui paraissent pertinents pour cette typologie de projets, et concentrent leurs effors eux.

Dans l'exemple ci-dessous, l'organisation générale, le processus de construction et la documentation ont été considérés comme nécessaires et suffisants pour couvrir l'évaluation. Chacun est doté d'un poids, et d'une note. L'ensemble donne une moyenne générale que l'on retrouve sur la médaille. Il est possible bien sûr de zoomer dans le détail chacun des thèmes.

Nous pourrions pu ajouter d'autres thèmes comme Performance, Conception technique...
